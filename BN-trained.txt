COALESCING PROCESS:
When duplicate columns appeared (e.g., HIV_PREVALENCE_x, HIV_PREVALENCE_y):
  1. Convert both to numeric
  2. Combine using combine_first() method
  3. Result: Single canonical column name

Example:
  HIV_PREVALENCE_x: [0.1, 0.05, 0.08]
  HIV_PREVALENCE_y: [0.1, 0.05, 0.08]
  ↓
  HIV_PREVALENCE: [0.1, 0.05, 0.08]

         (TB_NOTIFICATION_RATE / 100000) × 7.5

                    2. BAYESIAN NETWORK ARCHITECTURE

NETWORK TOPOLOGY
────────────────────────────────────────────────────────────────

Total Nodes: 10
Total Edges: 9
Network Type: Directed Acyclic Graph (DAG)
Learning Method: Structure defined manually, Parameters learned via MLE

NODE DEFINITIONS:
─────────────────────────────────────────────────────────────────

NODE 1: Socioeconomic_Status
  ├─ States: {"Low"}
  ├─ Parents: None (root node)
  ├─ Children: Malnutrition
  ├─ Data Source: Dummy (not in dataset)
  ├─ Role: Contextual factor
  └─ Rationale: Social determinant of health

NODE 2: HIV_Status
  ├─ States: {"Low", "Medium"}
  ├─ Parents: None (root node)
  ├─ Children: Progression_Risk
  ├─ Data Source: HIV_PREVALENCE_CAT
  ├─ Role: Risk multiplier for TB progression
  └─ Rationale: HIV increases TB risk 10-100 fold

NODE 3: Malnutrition
  ├─ States: {"Low", "High"}
  ├─ Parents: Socioeconomic_Status
  ├─ Children: Progression_Risk
  ├─ Data Source: MALNUTRITION_RATE_CAT
  ├─ Role: Nutritional immune status factor
  └─ Rationale: Malnutrition impairs immune response

NODE 4: TPT_Coverage
  ├─ States: {"Low", "High"}
  ├─ Parents: None (intervention lever)
  ├─ Children: Progression_Risk
  ├─ Data Source: TPT_COVERAGE_CAT
  ├─ Role: Policy intervention variable
  └─ Rationale: TPT prevents LTBI→TB progression

NODE 5: Latent_Burden (LTBI Prevalence)
  ├─ States: {"Low", "Medium", "High"}
  ├─ Parents: None (population epidemiology)
  ├─ Children: Active_TB
  ├─ Data Source: LTBI_PREVALENCE_CAT
  ├─ Role: Core disease state variable
  └─ Rationale: Larger LTBI pool → more active TB cases

NODE 6: Progression_Risk
  ├─ States: {"Low", "Medium", "High"}
  ├─ Parents: HIV_Status, Malnutrition, TPT_Coverage
  ├─ Children: Active_TB
  ├─ Data Source: PROGRESSION_RISK_CAT (derived)
  ├─ Role: Intermediate risk state
  └─ Rationale: Aggregates multiple risk factors

NODE 7: Facility_Distance
  ├─ States: {"Near", "Far"}
  ├─ Parents: None (infrastructure factor)
  ├─ Children: Active_TB
  ├─ Data Source: FACILITY_DISTANCE_CAT
  ├─ Role: Healthcare access barrier
  └─ Rationale: Distance affects detection and treatment

NODE 8: GeneXpert_Availability
  ├─ States: {"Available", "Limited"}
  ├─ Parents: None (infrastructure factor)
  ├─ Children: Active_TB
  ├─ Data Source: GENEXPERT_AVAILABILITY_CAT
  ├─ Role: Diagnostic technology access
  └─ Rationale: Improved diagnostics → better detection

NODE 9: Active_TB
  ├─ States: {"Low", "High"}
  ├─ Parents: Latent_Burden, Progression_Risk, Facility_Distance, GeneXpert_Availability
  ├─ Children: Treatment_Cost
  ├─ Data Source: ACTIVE_TB_INCIDENCE_CAT
  ├─ Role: Main outcome - disease progression state
  └─ Rationale: Central to model - what we want to predict

NODE 10: Treatment_Cost
  ├─ States: {"Low", "Medium", "High"}
  ├─ Parents: Active_TB
  ├─ Children: None (leaf node)
  ├─ Data Source: ESTIMATED_TREATMENT_COST_CAT
  ├─ Role: Economic outcome
  └─ Rationale: Impact of disease progression on costs

EDGE DEFINITIONS (Conditional Dependencies)
─────────────────────────────────────────────────────────────────

EDGE 1: Socioeconomic_Status → Malnutrition
  Direction: Socioeconomic_Status influences Malnutrition
  Meaning: Poorer socioeconomic status increases malnutrition risk
  CPD Type: P(Malnutrition | Socioeconomic_Status)
  States: Low SES → Higher malnutrition probability

EDGE 2: HIV_Status → Progression_Risk
  Direction: HIV_Status influences Progression_Risk
  Meaning: HIV+ status increases TB progression risk
  CPD Type: P(Progression_Risk | HIV_Status)
  States: High HIV → Higher progression risk

EDGE 3: Malnutrition → Progression_Risk
  Direction: Malnutrition influences Progression_Risk
  Meaning: Malnutrition weakens immune response
  CPD Type: P(Progression_Risk | Malnutrition)
  States: High malnutrition → Higher progression risk

EDGE 4: TPT_Coverage → Progression_Risk
  Direction: TPT_Coverage influences Progression_Risk
  Meaning: TPT reduces TB progression risk
  CPD Type: P(Progression_Risk | TPT_Coverage)
  States: Low TPT coverage → Higher progression risk
  Key Insight: PREVENTION LEVER - Can directly reduce risk

EDGE 5: Latent_Burden → Active_TB
  Direction: Latent_Burden influences Active_TB
  Meaning: More LTBI → more active TB cases
  CPD Type: P(Active_TB | Latent_Burden)
  States: High LTBI → Higher active TB probability
  Key Insight: CORE MODEL - LTBI→Active TB progression

EDGE 6: Progression_Risk → Active_TB
  Direction: Progression_Risk influences Active_TB
  Meaning: Higher risk factors → more progression
  CPD Type: P(Active_TB | Progression_Risk)
  States: High risk → Higher active TB probability

EDGE 7: Facility_Distance → Active_TB
  Direction: Facility_Distance influences Active_TB
  Meaning: Far facilities → worse detection/access
  CPD Type: P(Active_TB | Facility_Distance)
  States: Far → Harder to detect/treat (may appear as more cases)

EDGE 8: GeneXpert_Availability → Active_TB
  Direction: GeneXpert_Availability influences Active_TB
  Meaning: Better diagnostics → better detection
  CPD Type: P(Active_TB | GeneXpert_Availability)
  States: Limited → More undetected active TB

EDGE 9: Active_TB → Treatment_Cost
  Direction: Active_TB influences Treatment_Cost
  Meaning: More active TB → higher costs
  CPD Type: P(Treatment_Cost | Active_TB)
  States: High Active_TB → Higher costs
  Key Insight: OUTCOME - Cost consequence of disease progression

NETWORK STRUCTURE SUMMARY
─────────────────────────────────────────────────────────────────

Tier 1 (Root Causes):
  • Socioeconomic_Status → Malnutrition
  • HIV_Status (root)
  • TPT_Coverage (root, intervention lever)

Tier 2 (Risk Aggregation):
  • Progression_Risk (depends on: HIV, Malnutrition, TPT)
  • Latent_Burden (root, epidemiology)
  • Facility_Distance (root, infrastructure)
  • GeneXpert_Availability (root, infrastructure)

Tier 3 (Main Outcome):
  • Active_TB (depends on: Latent_Burden, Progression_Risk,
                           Facility_Distance, GeneXpert_Availability)

Tier 4 (Economic Outcome):
  • Treatment_Cost (depends on: Active_TB)

MODEL ESTIMATION PROCESS
────────────────────────────────────────────────────────────────

Algorithm: Maximum Likelihood Estimation (MLE)

For each node in topological order:
  1. Get all parent state combinations observed in training data
  2. Count frequency of each parent combination
  3. For each (parent combination, child state) pair:
       Count occurrences in training data
       Probability = Count / Total occurrences of parent combo
  4. Apply Laplace smoothing for unobserved combinations
       P(child | parent) = (count + 1) / (total + num_states)

Example: Estimating P(HIV_Status)
  ─────────────────────────────────
  Training data:
    KAMPALA:  Medium
    MUKONO:   Low
    WAKISO:   Medium

  Counts:
    Low:     1
    Medium:  2
    Total:   3

  MLE Probabilities:
    P(Low) = 1/3 = 0.333
    P(Medium) = 2/3 = 0.667

from pgmpy.models import DiscreteBayesianNetwork
from pgmpy.estimators import MaximumLikelihoodEstimator

# 1. Define network structure (edges)
model = DiscreteBayesianNetwork([
    ('Socioeconomic_Status', 'Malnutrition'),
    ('HIV_Status', 'Progression_Risk'),
    ('Malnutrition', 'Progression_Risk'),
    ('TPT_Coverage', 'Progression_Risk'),
    ('Latent_Burden', 'Active_TB'),
    ('Progression_Risk', 'Active_TB'),
    ('Facility_Distance', 'Active_TB'),
    ('GeneXpert_Availability', 'Active_TB'),
    ('Active_TB', 'Treatment_Cost'),
])

# 2. Prepare training data
train_df = merged_bn_categories[[
    'Socioeconomic_Status',
    'HIV_Status',
    'Malnutrition',
    'TPT_Coverage',
    'Facility_Distance',
    'GeneXpert_Availability',
    'Latent_Burden',
    'Progression_Risk',
    'Active_TB',
    'Treatment_Cost',
]].dropna().copy()

# 3. Convert to categorical
for col in train_df.columns:
    train_df[col] = train_df[col].astype(str).astype('category')

# 4. Estimate CPDs using MLE
estimator = MaximumLikelihoodEstimator(model, train_df)
for node in model.nodes():
    cpd = estimator.estimate_cpd(node)
    model.add_cpds(cpd)

# 5. Validate model
model.check_model()  # Checks DAG validity and CPD consistency

from pgmpy.inference import VariableElimination

# Create inference engine
infer = VariableElimination(model)

# Query 3: MAP (Most Probable Explanation)
map_estimate = infer.map_query(['Active_TB'])
